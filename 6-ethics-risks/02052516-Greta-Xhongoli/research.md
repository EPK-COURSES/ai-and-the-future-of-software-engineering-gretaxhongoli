Introduction 

Artificial Intelligence has made transformation on many areas such as healthcare, finance, transportation, education and also communication. One of the fields experiencing the greatest change is software engineering. Modern AI tools can now generate software code, explain programming concepts, detect bugs, and create complete applications with simple steps. The improvement of AI into software development has created significant benefits: software is produced faster, improved developer productivity, reduced workload, and easier access to programming assistance.Many companies nowadays use AI-powered coding applications to accelerate development and reduce costs. These systems are especially useful for repetitive programming.
On the other side , AI-assisted software development also creates important ethical and technical concerns. AI systems do not understand the software how the human do. Instead, they predict outputs using structures learned from massive data collections. Because of this limitation, AI-generated code may contain errors, vulnerabilities, or misleading solutions.
Based to studies on ethical risk in artificial intelligence , developers and organizations have ethical responsibilities toward users affected by AI systems. Ethical risks take place when stakeholders fail to fulfill these responsibilities to others.   This issue becomes really important when software engineers begin relying completely on AI-generated code without fully understanding it. If engineers stop understanding the code they ship, software systems may become insecure, unreliable, and difficult to maintain.
This paper explores the most important risks, limitations, and ethical concerns associated with AI-assisted software engineering while examining the long-term effects of overdependence on AI technologies.

AI-Assisted Software Development

AI-assisted software development refers to the use of artificial intelligence systems to support programmers in creating software , testing, debugging, and maintenance. Popular AI coding tools include: GitHub Copilot,  ChatGPT, Amazon CodeWhisperer and Replit Ghostwriter. These tools can: generate codes, complete functions, explain algorithms, find errors,and fasten development. AI systems are trained on huge collections of source code, online repositories, technical forums, and software documentation. By analyzing these models, they generate responses based on statistical predictions rather than true understanding. Research on AI-driven software development highlights that AI tools improve productivity and efficiency but also create ethical challenges related to accountability, privacy, and responsibility.  Although AI can significantly assist developers, software engineering still requires:  logical reasoning, problem-solving ability, security awareness, and ethical judgment.
Without human oversight, AI-generated software may introduce serious risks into real-world systems.
